## 建议使用方式

如果你正在准备面试，可以按下面顺序看：

1. 先看初级篇，确保基础概念说得准
2. 再看中级篇，补齐工程实践和现代 C++ 能力
3. 最后看高级篇，用于回答深入追问和区分度问题

## 推荐学习路径

### 第一轮

目标是“能回答”：

- 知道概念定义
- 知道为什么这样设计
- 能写出最小示例

### 第二轮

目标是“能讲清 tradeoff”：

- 什么时候用
- 什么时候不要用
- 性能、可维护性、异常安全上的区别

### 第三轮

目标是“能从面试题过渡到工程问题”：

- 如何避免内存泄漏
- 如何设计所有权模型
- 如何处理并发访问
- 如何写出可维护的泛型代码

## 常见面试考察维度

- 语言基础是否扎实
- 是否理解资源管理和对象生命周期
- 是否掌握现代 C++ 的核心工具
- 是否能把“语法知识”转成“工程判断”
- 是否有调试、性能和并发方面的意识

## 后续可以继续扩展的方向

- 调试与性能分析专题
- 系统设计专题
